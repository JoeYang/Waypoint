Problem one · Blocking

One unanswered question stalls the whole run.

The agent reaches a fork it isn't allowed to decide — which database, whether to delete a file, how to handle an edge case — and it does the only thing the terminal lets it do: it stops and waits for you.

Everything queued behind that question waits too, even the dozens of tasks that have nothing to do with it. A three-hour run becomes a three-hour run plus however long it took you to glance at your terminal.

Problem two · Context

Re-entering a session means
rebuilding context.

You step away, the agent keeps going, and when you come back the only record of what happened is a wall of scrollback. Where did it land? What did it decide while you were gone? What is it doing right now?

Answering those questions takes minutes of reading every time — and that re-entry cost is what makes it hard to jump in and out of a long session.

How it works

Two moves: park the decision, map the work.

01

Park the decision

Instead of blocking in the terminal, the agent writes the decision to Waypoint — question, options, its recommendation — and keeps working on everything not blocked by it.

02

Map the work

Every task, its status, and a live "you are here" marker render as a plan you can read in seconds — so jumping back in never means scrolling the logs.

Anatomy of a proposal.

Which ORM should the data layer use?

Parked 12m ago·agent continued on 5 unblocked tasks·src/db/index.ts
Why this came up
About to write the first six repository modules. The ORM choice shapes every query, the typing story, and the migration tooling — so it's worth one decision now rather than a refactor later.
Options & tradeoffs
Prisma
+Best-in-class DX, typed client
−Heavy runtime, codegen step
Agent recommends
Drizzle
+Thin, SQL-first, fully typed
−Younger ecosystem
Knex
+Battle-tested, flexible
−No types, manual mapping
If you defer: the agent keeps building against a thin Repository interface and stubs the implementation. Switching later costs about one file.

Acting on it

Approve at a glance, or redirect the agent.

Approve

One tap takes the recommendation. The agent picks it up and the parked task goes green — no restart, no re-prompt.

Pick another

Prefer a different option? Choose it directly. The agent proceeds with your call instead of its own.

Comment & redirect

Neither fits? Leave a note. It threads straight back to the agent as direction — a conversation, not a dead end.

Use Drizzle, but keep raw SQL migrations in db/migrations — I want them reviewable.
↩ sends to agent · threads on the decision

Every path resolves the decision and unblocks its task — whether you took ten seconds or wrote a paragraph. The agent never sat idle waiting to find out which.
